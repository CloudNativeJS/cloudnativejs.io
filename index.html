Create Cloud Native Node.js Applications.

CloudNativeJS is a community project that provides assets and tools to help you to build best-practice Cloud Native Node.js applications and deploy them to Docker and Kubernetes based clouds.

Choose your Node.js modules...

CloudNativeJS’s module insights constantly runs testing, verification and analysis against all current LTS versions of Node.js, across a wide set of operating systems and architectures. Module insights additionally checks and reports code coverage and verifies the license contained in the module and all of its dependencies.

This allows application developers to easily and consistently view the state of a range of modules across the platforms that they use, and for the CloudNativeJS community to work with module owners where there are test failures to get those resolved quickly.

Add health checking...

The “cloud-health” and “cloud-health-connect” repositories provide NPM modules for managing and reporting the lifecycle of your application through readiness and liveness endpoints. These are used by Kubernetes to determine whether your application is ready to receive requests, whether it is no-longer “live” and should be automatically restarted, and to handle graceful shutdown when it is requested to stop.

Track your requests...

The “appmetrics-zipkin” repository provides an NPM module for instrumenting your application with OpenTracing based request tracking, allowing you to monitor requests across microservces and applications. The tracking data is then sent to a deployment of the Zipkin open source request tracking project.

Power your metrics...

The “appmetrics-prometheus” repository provides an NPM module for collecting metric data from your application. The metrics gathered range from resource metrics such as CPU and memory, to application level metrics such as HTTP request responsiveness. These are then exposed as an application endpoint, and through a configuration in the Helm charts, the metrics are automatically collected by a deployment of the Prometheus open source monitoring project.

Build your Docker image...

The “docker” repository provides a set of template Dockerfiles that implement the best-practices for building development, debug, test and production Docker images for your applications. These can easily be modified and applied to any Node.js application.

Deploy to Kubernetes.

The “helm” repository provides a template Helm chart that simplifies the process of taking a Docker image of your application and deploying it to Kubernetes along with configuration for replicas and autoscaling. Additionally the templates provide configuration for using liveness and readiness based Health Checking, Prometheus metrics, OpenTracing with Zipkin, and support for using additional cloud-native technologies like Istio.
